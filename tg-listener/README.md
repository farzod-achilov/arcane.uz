# tg-listener — автозачисление депозитов из карточного Telegram-бота

Маленький сервис на GramJS (MTProto). Логинится **вашим личным аккаунтом**,
читает чат с карточным ботом (уведомления вида «Perevod na kartu, +85 000.00 UZS,
\*\*\*0133, 09.07.26 13:42») и пересылает текст в webhook сайта
`/api/deposit/sms-hook`. Дальше сайт сам находит заявку по уникальной сумме
и пополняет баланс пользователя.

Почему не наш обычный бот: Telegram Bot API не отдаёт ботам сообщения других
ботов — читать уведомления может только клиент-аккаунт (MTProto).

## Настройка (один раз)

1. **API-ключи Telegram**: https://my.telegram.org → API development tools →
   создать приложение → скопировать `api_id` и `api_hash`.

2. ```bash
   cd tg-listener
   cp .env.example .env     # вписать TG_API_ID, TG_API_HASH
   npm install
   npm run login            # телефон → код → печатает TG_SESSION
   ```
   Строку `TG_SESSION=...` добавить в `.env`.

3. В `.env` указать:
   - `SOURCE_CHATS` — юзернейм карточного бота без `@` (открыть профиль бота,
     который присылает уведомления, скопировать username). Можно несколько
     через запятую.
   - `WEBHOOK_URL` — `https://arcane.com.uz/api/deposit/sms-hook`
   - `SMS_HOOK_SECRET` — тот же, что в `.env.local` сайта.

4. Запуск на сервере (PM2):
   ```bash
   pm2 start index.js --name arcane-tg-listener --cwd /var/www/arcane/tg-listener
   pm2 save
   pm2 logs arcane-tg-listener
   ```

## Проверка

Сделайте тестовый перевод самому себе на карту из пула (или дождитесь любого
уведомления от карточного бота) — в логах появится строка
`→ webhook 200 {"ok":true,...}`. Если заявки с такой суммой нет, придёт
предупреждение в админский Telegram-чат.

## Важно

- Сессия (`TG_SESSION`) даёт полный доступ к аккаунту — хранить только в `.env`
  на сервере, никуда не коммитить.
- Уведомления с временем операции старше 90 минут автоматически НЕ зачисляются —
  уходят админу на ручную проверку (защита от повторов и старых пересылок).
- Если listener лежал (рестарт сервера) — пропущенные за это время переводы
  не переиграются автоматически: их видно в чате карточного бота, зачислить
  можно вручную в /admin/deposits.
